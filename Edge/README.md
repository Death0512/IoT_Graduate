# Edge Node – AI Processing Unit for IoT_Graduate

## What the Edge Node Does
- **Vehicle detection** (YOLO11s)  
- **Speed measurement** using homography (real‑world metres → km/h)  
- **License plate detection & recognition** (Vietnamese plates)  
- **Overspeed alerts** (snapshots + WebSocket notifications)  
- **Multi‑output**: HDMI display, MP4 file, WebRTC stream  

It listens to MQTT commands from the Master to dynamically add/remove cameras during load balancing.

## Hardware & Software Requirements
- NVIDIA Jetson (Orin NX/Nano, AGX Orin) with **JetPack 6.x** and **DeepStream SDK 7.1**  
- DeepStream dependencies (already installed with JetPack)  
- Python 3.10

## Setup on a Jetson

```bash
cd ~/IoT_Graduate/Edge
chmod +x setup_system.sh
./setup_system.sh       # installs system packages, sets up GStreamer
pip3 install -r requirements.txt
```

## Calibration (Homography & ROI)

Use the provided GUI tool to define the measurement area:

```bash
python3 speed_gui.py
```

1. **Sources** tab: add your RTSP stream or a video file.  
2. **Calibration** tab: click four points on the road that form a rectangle.  
3. Enter real‑world width (m) and length (m).  
4. Save – creates `configs/points_*.yml` and updates `configs/config_nvdsanalytics.txt`.

## Running the Health Agent (required for load balancing)

```bash
export MQTT_BROKER_HOST="192.168.1.100"   # IP of Master node
export NODE_ID="edge_orin_01"
python3 health_agent.py
```

The agent publishes GPU/CPU/RAM usage and current FPS every 2 seconds to `edge/status/<NODE_ID>`.

## Running the DeepStream Pipeline

### Common arguments

| Argument | Values | Description |
|----------|--------|-------------|
| `--backend` | `python`, `cpp` | Python backend (easier debug) or C++ backend (max performance) |
| `--source` | URL or file path | e.g. `rtsp://...`, `video.mp4` |
| `--mode` | `display`, `file`, `webrtc` | Output sink |
| `--homo` | path to YAML | Homography calibration file (from speed_gui.py) |
| `--cfg` | path to config | Camera‑specific config (FPS, resolution) – optional for WebRTC mode |

### Mode Examples

#### Display (HDMI)
```bash
python3 main.py --backend python \
  --source rtsp://192.168.1.200:8554/cam_01 \
  --mode display \
  --homo configs/points_1.yml
```

#### File (MP4)
```bash
python3 main.py --backend cpp \
  --source rtsp://192.168.1.200:8554/cam_01 \
  --mode file --output logs/result.mp4 \
  --homo configs/points_1.yml
```

#### WebRTC (browser)
Start the signaling server first (see `../Master/webrtc/README.md`).  
Then:
```bash
python3 main.py --backend python \
  --source rtsp://192.168.1.200:8554/cam_01 \
  --mode webrtc \
  --server 192.168.1.100 --port 8080 --room traffic_lane \
  --cfg configs/config_cam.txt
```

Open `http://192.168.1.100:8080/?room=traffic_lane` in any browser.

## Dynamic Camera Control (Load Balancing)

While the pipeline is running, it listens for MQTT commands from the Master on `edge/command/<NODE_ID>`:

- `ADD <rtsp_url>` – adds a new stream to the pipeline (without restarting)  
- `REMOVE <rtsp_url>` – removes a stream  

These commands are processed by `mqtt_subscriber.py` inside the pipeline. You can see log messages confirming the addition/removal.

## Configuration Files (important)

| File | Purpose |
|------|---------|
| `configs/config_infer_primary_yolo11.txt` | YOLO11 vehicle detector settings |
| `configs/config_infer_secondary_lpd.txt` | License plate detector |
| `configs/config_infer_secondary_lpr.txt` | OCR model for plates |
| `configs/config_nvdsanalytics.txt` | ROI polygon (auto‑generated by speed_gui.py) |
| `configs/config_tracker_NvDCF_perf.yml` | Vehicle tracker algorithm |
| `speedflow/settings.py` | Speed limits, validation thresholds, paths |

## Troubleshooting

- **No video / black screen** – check RTSP URL and network connectivity.  
- **Low FPS** – try `--backend cpp` or reduce inference batch size.  
- **MQTT commands not received** – verify `MQTT_BROKER_HOST` and that mosquitto is running on the Master.  
- **Homography wrong speed** – recalibrate with `speed_gui.py`; ensure the four source points form a rectangle on the road plane.

For further details, see the full code documentation in `speedflow/` and `main.py`.
